# PRD MVP Framing

This document supplies the rubric the PRD Builder uses to scope a minimum viable product (MVP) or thinnest viable slice. The rubric prose below is repository-original Microsoft content. The Lean Startup and SVPG sources that inform the framing are listed in a cite-only bibliography; no upstream prose is redistributed.

## When to apply

Apply this reference during the PRD Discover and Scope phases when:

* Deciding what belongs in the first releasable slice versus a later increment.
* Justifying In-Scope and Out-of-Scope entries in the canonical PRD template.
* Pressure-testing a scope proposal against learning goals and risk.

## Thinnest-viable-slice rubric (repository-original)

A candidate MVP slice passes when every check below is satisfied. Record the answers alongside the Scope section.

1. **Learning goal is explicit.** The slice names the single most important assumption it tests. A slice that tests nothing is a release, not an MVP.
2. **Delivers standalone value.** A real user can complete one end-to-end job with the slice, even if narrow.
3. **Smallest credible scope.** No requirement is included that is not needed to deliver the value or test the assumption.
4. **Measurable outcome.** The slice defines an observable signal (a metric or a qualitative threshold) that tells the team whether the assumption held.
5. **Reversible or contained.** The slice limits blast radius — a feature flag, a pilot cohort, or a bounded rollout — so a failed assumption is cheap to unwind.
6. **Out-of-scope is deliberate.** Each excluded capability is listed with a one-line reason, not omitted silently.

Scoring guidance: a slice that fails check 1 or check 3 is not yet an MVP and should be re-cut before requirements are committed.

The sources below inform the MVP framing. They are cited by name only; the rubric above is written from scratch and reproduces no upstream text.

### Eric Ries — *The Lean Startup*

* **Author / source** — Eric Ries, *The Lean Startup* (Crown Business, 2011).
* **URL** — [https://theleanstartup.com/](https://theleanstartup.com/)
* **Why the PRD Builder cites it** — Origin of the build–measure–learn loop and the "minimum viable product" framing used to justify thinnest-slice scoping. Referenced by name as an external depth pointer; never paraphrased clause-by-clause.

* **URL** — [https://www.svpg.com/inspired-how-to-create-products-customers-love/](https://www.svpg.com/inspired-how-to-create-products-customers-love/)
